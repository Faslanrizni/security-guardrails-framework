# Security Policy for [REPO_NAME]

## Supported Versions

| Version | Supported |
|---------|-----------|
| 1.x     | ✅ Security patches |
| < 1.0   | ❌ No support |

## Reporting a Vulnerability

**DO NOT CREATE PUBLIC ISSUES**

1. Email security@company.com with:
   - Subject: `[SECURITY] [REPO_NAME] Brief description`
   - Description of the issue
   - Steps to reproduce
   - Potential impact

2. Expect acknowledgment within 24h
3. We'll coordinate fix and disclosure

## Security Commitments
- Critical issues fixed within 7 days
- Security patches backported to last 2 versions
